Project Case Study

Palma Project

Palma Project is built around two connected tracks: the creation of the Palma-1.0 dataset and the development of a dedicated palm encyclopedia web platform. The objective is to provide a reliable, structured, and research-ready knowledge base for global palm biodiversity.

Project Scope

  • Track 1 - Palma-1.0 Dataset: build and publish a large structured palm database combining taxonomy, morphology, geography, and fruit trait metadata.
  • Track 2 - Encyclopedia Platform: build a public web experience to browse, interpret, and reuse palm data with a clean and usable research interface.

The platform and the dataset are designed as one ecosystem: data quality drives product quality, and product clarity increases data accessibility.

Dataset Overview

Palma-1.0 is a global database centered on the Arecaceae family and built from PalmTraits 1.0, then enriched with additional biodiversity and media sources.

  • 2,557 species documented across global tropical and subtropical regions.
  • 181 genera represented across the family breadth.
  • 120+ countries covered through distribution data.
  • 35 variables across taxonomy, growth form, morphology, fruit traits, and geography.

Data Sources

  • PalmTraits 1.0: base layer for global palm trait and ecological information.
  • GBIF: occurrence and taxonomic support with large-scale biodiversity records.
  • iNaturalist: citizen-science observations and visual references.
  • POWO: accepted taxonomy backbone from Royal Botanic Gardens, Kew.
  • Wikimedia Commons: CC-licensed image references and attributions.

Public data access and supporting pages: palma-encyclopedia.com/data.

Methodology

The production workflow follows a full scientific-style pipeline, from source collection to curated publication.

  • Collection: ingest PalmTraits 1.0 and enrich records via external source integrations.
  • Taxonomic Harmonization: normalize taxonomy against POWO to resolve inconsistencies and synonym issues.
  • Validation & Cleaning: detect outliers, correct anomalies, and verify problematic records against literature.
  • Geospatial Enrichment: integrate environmental and location context to improve ecological interpretation.
  • Analysis & Structuring: organize fields for practical biodiversity exploration and downstream analysis.
  • Publication: release in multiple formats for broad compatibility across research and engineering workflows.

Data Structure Highlights

  • Taxonomy: scientific names, genera, species epithet, tribe, and subfamily references.
  • Native Zone & Names: regional origin and multilingual common names (EN/FR/ES).
  • Growth & Habit: climbing, stem architecture, height/diameter limits, and canopy position.
  • Leaf Variables: armature, counts, and dimensional metrics for blade, rachis, and petiole.
  • Fruit Traits: length, width, shape, color descriptors, and conspicuousness categories.
  • Media Metadata: photo links and attribution references where available.

Distribution Formats & Access

Palma-1.0 is distributed in researcher-friendly formats:

  • CSV
  • JSON
  • Plain text

Official dataset repository: huggingface.co/datasets/kitsuiwebster/Palma-1.0.

Web Platform Architecture

  • Frontend: Angular single-page application for a responsive encyclopedia experience.
  • UI Layer: SCSS-driven interface and structured page modules focused on readability and hierarchy.
  • Data Presentation: species-centric views designed to surface taxonomy, traits, media, and distribution context.
  • Deployment: Netlify-hosted static delivery for stable and fast global access.

The product is intentionally front-end focused: straightforward hosting, clear navigation, and strong content legibility.

Research Utility & Positioning

  • Supports exploration of global palm diversity at species and genus levels.
  • Combines functional traits and geographic context in one reusable format.
  • Improves accessibility for education, biodiversity analysis, and project prototyping.
  • Provides a base layer that can be extended in future dataset iterations.
