Palma Project

Palma Project is a two-layer biodiversity product: a high-value research dataset (Palma-1.0) and a public encyclopedia interface built to make palm knowledge easier to explore, validate, and reuse at scale.

Executive Scope

Palma Project is intentionally split into two professional deliverables:

  • Palma-1.0 Dataset: a global palm database designed for structured analysis, publication, and reproducible downstream usage.
  • Palma Encyclopedia Web App: a front-end product that makes palm taxonomy and trait information accessible to non-specialists and technical users alike.

This split keeps responsibilities clean: data engineering focuses on scientific coherence and normalization, while product engineering focuses on navigation, understanding, and trust in the published knowledge.

Dataset Snapshot

  • Family Scope: Arecaceae (global palm diversity).
  • Species Coverage: 2,557 species.
  • Genus Coverage: 181 genera.
  • Geographic Breadth: 120+ countries across tropical and subtropical regions.
  • Variable Count: 35 core variables spanning taxonomy, growth, morphology, geography, and fruit descriptors.
  • Publication Formats: CSV, JSON, and plain text.

Source Stack

Palma-1.0 starts from PalmTraits 1.0 and is then enriched with high-value biodiversity and taxonomy providers:

  • PalmTraits 1.0: foundational trait/ecology base.
  • GBIF: large-scale occurrence and taxonomic context.
  • iNaturalist: observation and visual support from a citizen-science network.
  • POWO (Plants of the World Online): taxonomy authority for accepted names and harmonization.
  • Wikimedia Commons: reusable visual documentation with explicit attribution metadata.

The design goal is source complementarity: taxonomy consistency from POWO, occurrence depth from GBIF and iNaturalist, and image traceability via Wikimedia metadata.

Methodology Deep Dive

The methodology follows a strict staged pipeline aligned with the data page and dataset documentation.

  • 1) Baseline Ingestion: import PalmTraits 1.0 entities and normalize primary fields.
  • 2) External Enrichment: query GBIF, iNaturalist, and POWO layers to expand taxonomic and ecological context.
  • 3) Taxonomic Harmonization: enforce accepted naming references, resolve synonymies, and reduce naming drift.
  • 4) Validation & Cleaning: detect outliers, reconcile inconsistent records, and review edge cases against scientific references.
  • 5) Geospatial Enrichment: attach distribution-aware context and environmental framing where relevant.
  • 6) Structuring for Reuse: consolidate stable columns and semantic groupings for direct research usage.
  • 7) Multi-Format Publication: release in CSV/JSON/TXT with documentation and citation guidance.

The core principle is not only adding data, but preserving coherence between taxonomy, trait semantics, and distribution context. This improves reliability for both exploratory analysis and production reuse.

Schema Design

Palma-1.0 is organized into practical variable families rather than a flat undocumented table. This makes both manual exploration and programmatic parsing significantly cleaner.

  • Taxonomy: species full name, genus, species epithet, tribe, subfamily.
  • Native Zone & Common Names: native region fields plus multilingual naming (EN/FR/ES).
  • Growth & Habit: climbing behavior, stem architecture, armed/not armed state, size limits.
  • Leaf Morphology: armed status, max leaf count, blade/rachis/petiole dimensions.
  • Fruit Morphology: min/avg/max dimensions, categorical size, shape and color descriptors.
  • Visibility & Media: fruit conspicuousness and image references with attribution metadata.

This structure enables both descriptive usage (encyclopedia navigation) and analytical usage (trait filtering, aggregation, correlation exploration, and model-ready preprocessing).

Quality & Reliability Strategy

  • Consistency Checks: resolve conflicting taxonomy and malformed trait entries.
  • Outlier Review: identify improbable values and prioritize manual verification on suspicious records.
  • Semantic Normalization: align repeated descriptors to avoid fragmented categorical values.
  • Attribution Hygiene: preserve source references for images and externally enriched metadata.
  • Publication Stability: keep a documented and reproducible schema for future version upgrades.

Data Product Outputs

Palma-1.0 is published as a reusable data product, not as a one-off export. The outputs are designed for interoperability across scientific and engineering workflows.

  • CSV: spreadsheet-friendly and pipeline-friendly baseline exchange format.
  • JSON: direct integration for web/data APIs and scripting workflows.
  • TXT: lightweight text representation for simple processing and archival portability.

Official distribution: Hugging Face - kitsuiwebster/Palma-1.0.

Encyclopedia Frontend Architecture

The web platform is an Angular frontend focused on high legibility and structured content delivery. Architecture decisions prioritize clean navigation and predictable rendering over unnecessary complexity.

  • Framework Core: Angular + TypeScript for modular page composition.
  • UI Layer: SCSS and semantic HTML for maintainable style architecture.
  • Content Surfaces: species-oriented content and dedicated data/methodology context.
  • Hosting Strategy: Netlify deployment for stable static delivery and simple release operations.

The approach remains intentionally front-end centric, which keeps operational load low while preserving a professional and scalable presentation layer for a data-heavy domain.

How Dataset and App Reinforce Each Other

  • The dataset provides authoritative structure and measurable depth.
  • The app translates that structure into navigable and understandable pages.
  • The methodology page increases transparency for both technical and non-technical readers.
  • Hugging Face publication gives direct access for programmatic reuse and reproducibility.

In practice, this creates a full loop: collect and normalize data, publish it with clear framing, then expose it in a user-facing encyclopedia with research-grade references.

Use Cases

  • Biodiversity Research: trait exploration and cross-region species comparison.
  • Education: clear entry point for students and readers discovering palm diversity.
  • Data Engineering: normalized palm records reusable for dashboards and analytical tooling.
  • Conservation Context: structured basis for geographic and ecological interpretation.
